# Claude Code Hints for web-starter

## Conventions
- Pages are CMS-driven via Sanity. Don't create new routes as files in app/; use the page builder.
- New block types: schema → component → register in Sections.tsx
- Server-only code goes in src/lib/, components in src/components/
- Use `sanityFetch` for all Sanity reads (includes cache tags).
- Tailwind classes only; no inline styles.

## Stack
- Next.js 15 App Router
- Sanity GROQ queries
- Tailwind CSS with CSS variables (--brand-*)
- Vercel deployment

## Before you code
1. Check src/lib/types.ts for existing types
2. Look at existing block components for patterns (Hero, RichText, Cta)
3. Use resolveLink() for internal/external link resolution
